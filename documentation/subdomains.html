Nice — here’s a clear, practical, step-by-step guide for creating a MERN app (MongoDB, Express, React, Node) and exposing it on a subdomain on your machine (so you can visit something like app.localhost and api.localhost). 1) Overview (what we'll create)

  • Backend: Node + Express + Mongoose (runs on port 5000)

  • Frontend: React (Create React App) (runs on port 3000)

  • MongoDB: local or Docker

  • Subdomains (locally): app.localhost for frontend, api.localhost for backend

    • Clean: hosts file + nginx reverse proxy to forward app.localhost → 3000 and api.localhost → 5000 so you can omit ports.

Note: *.localhost sometimes is special in some environments. If you see unexpected browser behavior, use app.test / api.test or app.local instead; same steps apply.

8) Clean approach — nginx reverse proxy (no ports in URL)

This forwards http://app.localhost → localhost:3000 and http://api.localhost → localhost:5000. Requires nginx running and listening on port 80.

  1. Install nginx (if not already).

  2. Create nginx site configuration (example for linux /etc/nginx/sites-available/mern_local):

                                    

                                        

                                            nginx

                                                    listen
                                                    80;
                                                    server_name
                                                        app.localhost;

                                                    location
                                                        / {
                                                    proxy_pass
                                                        http://127.0.0.1:3000;
                                                    proxy_set_header
                                                        Host
                                                    $host;
                                                    proxy_set_header
                                                        X-Real-IP $remote_addr;
                                                    proxy_set_header
                                                        X-Forwarded-For $proxy_add_x_forwarded_for;
                                                        }
                                                        }

                                                    server {
                                                    listen
                                                    80;
                                                    server_name
                                                        api.localhost;

                                                    location
                                                        / {
                                                    proxy_pass
                                                        http://127.0.0.1:5000;
                                                    proxy_http_version
                                                    1.1;
                                                    proxy_set_header

                                                    proxy_set_header

10) Useful dev improvements

  • Use concurrently (or npm-run-all) in root package.json to run frontend & backend with one command.

  • Add CORS config and cookie/session config if you plan to use authentication across subdomain (you may need to set sameSite and cookie domain properly).

  • For HTTPS locally, use mkcert to generate trusted certs and configure nginx to listen on 443 (optional, needed for OAuth or secure cookies).

  • If you want app.localhost to set cookies for localhost domain across subdomains, you will need to configure cookie domain properly and likely use HTTPS.

12) Troubleshooting checklist

  • Hosts file changes require DNS cache flush or browser restart. On macOS: sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder. On Windows: ipconfig /flushdns.

  • If React dev server refuses to bind to app.localhost, set HOST=app.localhost env when starting.

  • Confirm Express CORS origin exactly matches scheme + host (http://app.localhost).

  • If port 80 is in use, either stop that service or use nginx on another port and include port in URL.

  • If cookies not shared, ensure credentials: 'include' and server sets Access-Control-Allow-Credentials: true and cookie domain correct.
